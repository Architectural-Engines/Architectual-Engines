**Daily log***:
**4/07/2026 PART 1**:

**Main Takeaways on Model & Parameters**:

**Internal vs External Parameters**:

-Hyperparameters like layers, epsilon, state_dim only need to exist inside the model as self.layers, self.epsilon, etc.

-Only parameters that must be accessed outside (like theta_lat, branch_embed, device) should be returned from compute_global() or defined in main.

**Theta Handling**:

theta_d starts as a placeholder (torch.zeros(3)) and is transformed in compute_global() → theta_lat.

Only theta_lat (and optionally the original theta_d for reference) needs to be passed around outside the model.

Forward pass uses these transformed values.



**Branch Embed**:

-Needed externally for training/visualization; define and return it from compute_global().

-Avoid duplicating inside the model (self.branch_embed) unless necessary — decide where the logic belongs.


**Compute Global**:

**Return only what’s required externally**:

-return theta_lat, branch_embed, device

-Do not return hyperparameters like layers or epsilon if they’re only used internally.



**Forward & Main Flow**

**Main execution**:

-Define model hyperparameters.


-Create the model instance (model = ProbTrajectory5k(...)).


-Call compute_global() to get theta_lat, branch_embed, device.


-Pass those to train_on_synthetic() and other functions as needed.

Forward pass accesses internal hyperparameters via self.<param>; no need to pass them as arguments.



**Common Errors**:

missing 1 required positional argument → usually caused by trying to pass a parameter that already exists inside the model or by mismatched return values from compute_global().

TypeError with torch.randn() → make sure arguments are tuples (num_branches, embed_dim) not (num_branches, embed_dim) * 0.1 directly inside randn(). Multiply after creation.



**General Rule**:

Only pass/return what’s needed outside the model. Let the model store and handle its own internal hyperparameters.

Keep main clean: top-level orchestration of model, compute, and training loops.

Keep a clean seperation between logic, dimensions, self., internal/external parameters.
