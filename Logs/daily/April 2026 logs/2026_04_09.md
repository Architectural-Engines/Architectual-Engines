 **V1 PIPELINE FIX LOG — TIME / FEATURE / MODEL ALIGNMENT**

**Scope**:

-Time handling pipeline.

-Feature projection system.

-Model input contract (time_proj).

-Branch computation integrity.

-Compute/main separation.

**Core Issues Identified**:

-Time variable inconsistency.
 
**Mixed usage**:

t, t_traj, t_clean, and t_feat.

**Result**:

-Ambiguous execution contract across pipeline. 
-Feature dimension mismatch.
 
**Observed**:

-feats.shape = [T, 6].

-time_proj.in_features = 16.

**Error**:

-mat1 and mat2 shapes cannot be multiplied (20x6 and 16x16).

-Model layer redefinition in runtime.
 
**model.time_proj incorrectly re-initialized inside**:

-main.

-compute.

-Violates stable parameter initialization rule.

-Duplicate computation paths.

-branch_out computed in multiple locations.

**Result**:

-Inconsistent tensor state across execution paths

 -Missing pipeline dependency
 
-_mlp_branch() requires t_feat

-t_feat not guaranteed inside compute() in earlier versions

**Unsafe tensor construction pattern**:

**Deprecated usage**:

-torch.tensor(existing_tensor in Anneal V1 code)

**Replaced with**:

-tensor.detach().clone().float()

-Canonical Variable Definitions

-Raw trajectory time

-t_traj

**Output of trajectory generator**:

**Shape**:**[T]**

-Optional preprocessing

-t_clean = prep_trajectory(t_traj)

-Optional smoothing/filtering

-Must preserve trajectory alignment

-Engineered time features

-feats ∈ ℝ[T, 6]

-sin/cos frequency encoding

-Derived from t_traj

**Model time embedding**:

-t_feat = model.time_proj(feats)

-Shape: [T, embed_dim]

-Canonical input to model logic

**Branch output**:

-branch_out = _mlp_branch(branch_embed, t_feat)

-Single definition per forward/computation path

 **Architectural Contracts**
 
-Feature → projection contract

-feats.shape[-1] == model.time_proj.in_features

-Single source of truth rule

-t_feat defined exactly once per pipeline

-branch_out computed exactly once per pipeline

-Model initialization constraint

-model.time_proj = nn.Linear(feats_dim, embed_dim)
**Must occur in model initialization only
Must NOT be redefined in runtime**

**Ownership separation**:

-Layer Responsibility

-main input sanitization

-compute pipeline orchestration

-model learned transformations

**Deprecated Patterns**:

 -Runtime layer redefinition
 
-model.time_proj = nn.Linear(...)

 -Multiple naming for same tensor
 
-feats_proj

-t_feat (duplicate usage)

-mixed aliases across modules

 -Missing intermediate construction
 
_mlp_branch(..., t_feat) without guaranteed t_feat creation in same scope

**Stable Pipeline (Reference):
t_traj
  ↓
t_clean (optional)
  ↓
feats (sin/cos encoding, dim=6)
  ↓
t_feat = time_proj(feats) (dim=16)
  ↓
model forward
  ↓
branch_out (single computation)**

**Stability Outcome**:

-Eliminated dynamic model redefinition in runtime

-Standardized time variable roles

**Enforced single-path computation for**:

-t_feat

-branch_out

-Fixed feature dimension contract enforcement
