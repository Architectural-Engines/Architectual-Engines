**Session Notes** – [Anneal V1](https://github.com/Architectural-Engines/Architectual-Engines/tree/main/Models/ANNEAL%20V1) / ProbTrajectory5k Refactor

**Core Refactor Goals**:

Moved most of the computation logic outside of the model.

Restructured the forward method to act primarily as a logger / interface, keeping it lightweight (~30 lines).

Ensured stability while decoupling the model from heavy computation.


**Compute Function**:

Converted compute to a global function.

Removed all self. dependencies inside compute.

All parameters needed for computation (num_branches, embed_dim, state_dim, epsilon, layers) can be passed directly.

Returns key outputs: pos_super, unc, resolved_branches, branch_out.

Enables computation without loading the full model, useful for preprocessing or trajectory manipulation.


**Trajectory Handling**:

traj removed; replaced by t_clean (time) and pos_target_clean (positions).

prep_trajectory module applied to both t and pos_target for cleaning.

traj_idx retained for indexing but does not rely on traj.


**Forward Method**:

Now primarily handles logging and returns outputs.

Calls compute with cleaned trajectories when needed:

processed = compute(t_clean, pos_target_clean, theta_d, motion_type='circular', ...)

Forward no longer contains encoding, branch MLP, noise, aggregation, uncertainty computation, etc.

Checked optimizer and learning rate setup; ensured lr is a float, not a list.


**Model Parameters**

self. references pulled from forward/computation and passed explicitly to global functions.

Standardized parameter handling: num_branches, embed_dim, state_dim, epsilon, layers now accessible for global functions.

**Example**:

p_lat = P_map(P_d.to(device))

Allows global computation functions to operate independently of model instance.


**Training Loop Adjustments**

train_on_synthetic updated:

Removed redundant variables and simplified training loop (train_on_synthetic) with clear tensor handling.

theta_d passed explicitly; ensure matches forward/global call signature.

Cleaned tensor preparation inside loop:

Restricted the forward function in the model to act primarily as a logger / interaction point.


t_clean = prep_trajectory(t, kernel_size=3)
pos_target_clean = prep_trajectory(pos_target, kernel_size=3)

Epoch loop and optimizer now stable with the cleaned structure.


**Observations / Learnings**

Separating computation logic greatly reduces model size and forward complexity.

theta_d errors were due to mismatched argument passing, fixed by explicitly passing to global compute.

Global compute allows preprocessing without initializing full model.

Forward method now acts as a stable interface to call core math and logging.

All fixes done while preserving original functionality and output stability.
