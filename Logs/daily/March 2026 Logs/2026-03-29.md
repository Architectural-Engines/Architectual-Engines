**Trajectory Preprocessing**:

-**Smooth** trajectories to prevent instability.

-**Maintain** z-axis explicitly for 3D.

**Notes/Takeaways**:

-**Always** call prep_trajectory on each trajectory or position tensor before feeding it into the model.

-**ensure_traj3D** ensures consistent 3D shape.

-**smooth_tensor** prevents numerical spikes in any axis.

**Main Execution Patterns**:

-Training Loop (train_on_synthetic)

-Type-safe, handles multiple trajectories.

-**Ensures** all tensors are [T,3].

-**Fixed** lr handling and optimizer creation.

**Notes/Takeaway**:

-**Avoid** passing extra arguments like trajectory or kernel_size in multiple places.

-**Forward pass** expects cleaned [T,3] tensors.

**Common Pitfalls**:
**NoneType or list vs tensor**:

-**Always convert** to tensor and detach().clone() if already tensor.

**torch.tensor vs torch.Tensor**:

-**Use** torch.Tensor for type checks (isinstance), lowercase torch.tensor() to construct.

**Shape mismatches**:

-**Make sure** all axes are [T,3] before stacking, slicing, or smoothing.

**Duplicated/conflicting arguments**:

-**Avoid** passing learning rate (lr) multiple ways; define in one place.

**Kernel size**:

-**Only pass** into prep_trajectory and smooth_tensor. Do not pass to torch.tensor().

**Forward consistency**:

-Model forward should **always** receive cleaned, smoothed, 3D tensors.

**The current prep ensures**:

-**Z-axis** is explicitly maintained (no **approximation**)

-**Trajectories** can be stacked or projected safely

**Summary / Takeaways**:

-**Fix all preprocessing**

-**Always** work with clean [T,3] tensors.

-**Smooth trajectories** to avoid spikes.

-**Forward pass** and **training** must only operate on cleaned tensors.

-Keep track of consistent variable naming (t, pos_target, trajectory_clean, etc.) to avoid confusion.

**3/29/2026 PART 2**:
**Training Pipeline**

**Execution Order (MOST IMPORTANT)**:
Inside train_on_synthetic:

-loop → clean inputs → define dependent vars → model call → loss → optimizer
Correct order:
**1.** Get raw data (t, pos_target)
**2.** Clean data (t_clean, pos_target_clean)
**3.** Define dependent variables (traj_input, theta_d)
**4.** Call model
**5.** Compute loss
**6.** Backprop + step

**Variable Scope Rules**:
-Variables created inside loop → must be used inside loop

Do NOT pass these into the function:

-pos_target_clean

-resolved_state

-branch_out

**They are**:
-either computed inside or returned from model

**Function Responsibility**:

**main Creates**: model, trajectories

**Calls**: train_on_synthetic(model, trajectories, epochs=300)

**train_on_synthetic Handles**:preprocessingn model calls, training loop

**forward Handles**: math, branching, returns outputs

**Model Call Contract**

**model expects**: model(t, P_d, traj, theta_d, motion_type)

**MUST provide**: t_clean → time, pos_target_clean → target / P_d ,traj_input → trajectory input,
theta_d → directional tensor

**Data Prep Rules**:
t (time): stays 1D → DO NOT use prep_trajectory

**use**: t_clean = t.clone().float()

**pos_target**: must be [T,3]

**use**: pos_target_clean = prep_trajectory(pos_target, kernel_size=3)

**Summary**:
Refactored training flow to properly separate data prep, model inputs, and forward outputs, resolving multiple argument, scope, and tensor errors.

**Standardized call**:
-train_on_synthetic(model, trajectories, epochs=300)

**Variable Scope Corrections**:
-pos_target_clean must be defined inside the training loop, not passed from main

Execution order (**Critical Fix**) Established strict pipeline:
