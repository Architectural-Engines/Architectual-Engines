***Models***

**Purpose**
This folder contains **core, reproducible model iterations**, including Anneal and future models. 
It acts as a stable archive for models that have been validated and can be consistently reproduced.

**Workflow**
1. **Experimental Phase**: New models start in the `experiments` folder.  
2. **Validation**: Once a model demonstrates reproducible behavior and stable outputs, it is moved into the `Models` folder.  
3. **Reference & Expansion**: The folder serves as the central hub for reliable model iterations,
   enabling comparison, study, and further experimentation.

**Contents**
- **Reproducible Models**: Anneal V1 and future validated models.  
- **Seed Generator**: Visual and numerical seeds used to probe stability and behavior.  
- **Supporting Scripts**: Minimal scripts for running experiments, visualizing outputs, and managing iterations.
ANNEAL V1 [^1.1a].
**Notes**
- Models here are intended for **study, experimentation, and reproducibility**.  
- Observers should explore outputs and logs conceptually; behavior is emergent and best understood through iteration.  
- The folder is intentionally **expandable**: new models can be added as they are validated.

**Takeaway**
The `Models` folder is a **living archive of validated model development**, providing a stable reference point while allowing ongoing conceptual exploration.
